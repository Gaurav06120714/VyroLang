# Glossary

| Term | Definition |
|---|---|
| **AST** | Abstract Syntax Tree — structured tree representation of parsed source code. |
| **Bytecode** | Compact instruction format (`.vbc`) executed by the VyroVM. |
| **Codegen** | Code generation — compiler stage that emits bytecode from the analyzed AST. |
| **Coroutine** | Cooperatively-scheduled lightweight task used by the async runtime. |
| **GC** | Garbage Collector — automatic memory reclamation in the VM heap. |
| **IR** | Intermediate Representation — form between AST and bytecode used for optimization. |
| **Lexer** | Tokenizer — converts source characters into tokens. |
| **Opcode** | A single VM instruction (e.g., `PUSH`, `LOAD`, `CALL`). |
| **Parser** | Builds an AST from tokens according to the grammar. |
| **Sandbox** | Isolated Docker environment with resource limits where untrusted code runs. |
| **Semantic Analyzer** | Validates types, scopes, names, and function signatures. |
| **vpm** | VyroPackageManager — dependency manager and registry client. |
| **VyroAI** | AI assistant for completion, review, and docs. |
| **VyroCloud** | Cloud execution and project storage. |
| **VyroCompiler** | The Vyro source-to-bytecode compiler. |
| **VyroDeploy** | Push-to-deploy application hosting. |
| **VyroIDE** | Browser-based IDE. |
| **VyroLang** | The Vyro programming language. |
| **VyroOS** | The system abstraction layer (FS/Process/Memory/Net). |
| **VyroVM** | The virtual machine that executes Vyro bytecode. |
| **`.vy`** | VyroLang source file extension. |
| **`.vbc`** | Vyro bytecode file extension. |
